PolyCop OpenAPI is a REST API for external developers. This documentation covers only endpoints authenticated with API Key + HMAC-SHA256 signatures.

The API includes wallets, orders, positions, copytrade, AFK strategies, and partner-only user provisioning/API key creation. It does not cover first-party Web APIs, Bearer-token login APIs, account management APIs, or security setup APIs.

## Base URL

All API requests are made over HTTPS to the following base URL:

```
https://test-eu.polycopy.fun/openapi/v1
```

Append the endpoint path to this base URL for every request. For example, to call the wallet endpoint, use `https://test-eu.polycopy.fun/openapi/v1/wallet`.

## Response Envelope

Every response uses the same JSON envelope, whether the request succeeds or fails. Check `code` first, then read the endpoint-specific `data` field.

```json
{
  "code": "OK",
  "message": "success",
  "data": { ... },
  "request_id": "req_202606011200000001"
}
```

<ResponseField name="code" type="string" required>
  A machine-readable status code. `OK` means the request succeeded. Any other value is an error, such as `UNAUTHORIZED` or `NOT_FOUND`.
</ResponseField>

<ResponseField name="message" type="string" required>
  A human-readable description of the outcome. On success this is typically `"success"`. On error it contains a short explanation of what went wrong.
</ResponseField>

<ResponseField name="data" type="object | null" required>
  The endpoint-specific payload. On error this field is usually `null`.
</ResponseField>

<ResponseField name="request_id" type="string" required>
  A unique request identifier. Include it when you contact support.
</ResponseField>

## Error Format

When a request fails, `code` contains a non-`OK` value and `data` is usually `null`. The `message` field gives a short explanation.

```json
{
  "code": "UNAUTHORIZED",
  "message": "API key signature is invalid",
  "data": null,
  "request_id": "req_202606011200000002"
}
```

Always check `code`, not only the HTTP status. Log `request_id` with failed requests.

## Key Constraints

Before you integrate, note these constraints:

<Note>
  **Custodial wallet only.** PolyCop OpenAPI operates exclusively with custodial wallets. You do not manage private keys directly.
</Note>

<Note>
  **HMAC-SHA256 authentication.** Every API request must include `X-API-Key`, `X-Timestamp`, `X-Nonce`, and `X-Signature`. Trading operations do not require 2FA. Withdrawal requests require 2FA.
</Note>

<Note>
  **Not Polymarket CLOB-compatible.** This API is not compatible with the Polymarket CLOB API. Existing Polymarket CLOB integrations cannot be pointed at this base URL without changes.
</Note>

<Note>
  **Polygon network only (v1).** The current version of the API supports only the Polygon network. Multi-network support is not available in v1.
</Note>
