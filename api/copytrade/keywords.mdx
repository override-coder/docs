---
title: "Copytrade Keyword Management — Add and Remove Keywords"
sidebarTitle: "Keywords"
description: "Add keywords to a copytrade config whitelist or blacklist using POST. Remove keywords by ID using DELETE. Use instead of PATCH for keyword changes."
---

Copytrade configurations support keyword filters that control which markets are eligible to be copied. A whitelist restricts copying to markets whose names match at least one keyword; a blacklist excludes markets whose names match any keyword. Use the endpoints on this page to add or remove keywords **after** a configuration has been created — do not use `PATCH /copytrade/configs/{config_id}` for keyword changes, as keyword fields are rejected on that endpoint.

<Note>
  To set keywords at configuration creation time, pass `keyword_whitelist` and `keyword_blacklist` in the `POST /copytrade/configs` request body. Those fields are only accepted during creation.
</Note>

---

## Add a Keyword

```
POST /copytrade/configs/{config_id}/keywords
```

Use this endpoint to add a single keyword to the whitelist or blacklist of an existing copytrade configuration.

### Required Scope

`copytrade:manage`

### Optional Header

| Header | Description |
|---|---|
| `Idempotency-Key` | A unique key (e.g. a UUID) you generate. Repeated requests with the same key return the original response without adding a duplicate keyword. |

### Path Parameters

<ParamField path="config_id" type="string" required>
  The unique identifier of the copytrade configuration to add a keyword to.
</ParamField>

### Request Body

<ParamField body="key_word" type="string" required>
  The keyword text to add to the filter list.
</ParamField>

<ParamField body="list_type" type="integer" required>
  Specifies which list to add the keyword to. Use `1` for the whitelist or `2` for the blacklist.
</ParamField>

<ParamField body="config_type" type="integer" required>
  Specifies the configuration type. Use `1` for an application config or `2` for a template config.
</ParamField>

### Example Request

```bash
curl -X POST "https://test-eu.polycopy.fun/openapi/v1/copytrade/configs/123/keywords" \
  -H "X-API-Key: pk_your_key_id" \
  -H "X-Timestamp: 1717200000" \
  -H "X-Nonce: abc123" \
  -H "X-Signature: <signature>" \
  -H "Content-Type: application/json" \
  -d '{"key_word": "election", "list_type": 1, "config_type": 1}'
```

### Example Response

```json
{
  "code": "OK",
  "message": "success",
  "data": {
    "key_word_id": 456,
    "key_word": "election",
    "list_type": 1,
    "config_type": 1,
    "config_id": "123"
  },
  "request_id": "req_202606011300000001"
}
```

## Remove a Keyword

```
DELETE /copytrade/configs/{config_id}/keywords/{key_word_id}
```

Use this endpoint to remove a keyword from a copytrade configuration's filter lists by its record ID.

### Required Scope

`copytrade:manage`

### Optional Header

| Header | Description |
|---|---|
| `Idempotency-Key` | A unique key (e.g. a UUID) you generate. Repeated requests with the same key return the original response without erroring. |

### Path Parameters

<ParamField path="config_id" type="string" required>
  The unique identifier of the copytrade configuration from which to remove the keyword.
</ParamField>

<ParamField path="key_word_id" type="integer" required>
  The integer record ID of the keyword to remove.
</ParamField>

<Note>
  `key_word_id` is the integer record ID returned in the keyword data (e.g. from the `POST /copytrade/configs/{config_id}/keywords` response), **not** the keyword text itself. Store this ID when adding keywords if you plan to remove them later.
</Note>

### Example Request

```bash
curl -X DELETE "https://test-eu.polycopy.fun/openapi/v1/copytrade/configs/123/keywords/456" \
  -H "X-API-Key: pk_your_key_id" \
  -H "X-Timestamp: 1717200000" \
  -H "X-Nonce: abc123" \
  -H "X-Signature: <signature>"
```

### Example Response

```json
{
  "code": "OK",
  "message": "success",
  "data": {},
  "request_id": "req_202606011300000002"
}
```
