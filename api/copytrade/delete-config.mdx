title: "DELETE /copytrade/configs/{config_id} — Disable Copytrade Config"

This endpoint disables a copytrade configuration. The configuration is **not permanently deleted** — it transitions to `disabled` status, stops copying trades, and remains queryable via `GET /copytrade/configs/{config_id}` and `GET /copytrade/configs`. You can inspect a disabled configuration's settings at any time. Requires the `copytrade:manage` scope. You may optionally supply an `Idempotency-Key` header to safely retry the request.

## Request

```
DELETE /copytrade/configs/{config_id}
```

### Required Scope

`copytrade:manage`

### Optional Header

| Header | Description |
|---|---|
| `Idempotency-Key` | A unique key (e.g. a UUID) you generate. Repeated requests with the same key return the original response without triggering the disable operation a second time. |

### Path Parameters

<ParamField path="config_id" type="string" required>
  The unique identifier of the copytrade configuration to disable.
</ParamField>

## Response

The response `data` field contains the full `CopytradeConfig` object with `status` set to `"disabled"`.

<ResponseField name="data" type="object">
  <Expandable title="CopytradeConfig fields">
    <ResponseField name="config_id" type="string">
      Unique identifier for this copytrade configuration.
    </ResponseField>

    <ResponseField name="target_address" type="string">
      The trader wallet address that was being followed.
    </ResponseField>

    <ResponseField name="tag" type="string">
      Human-readable label for this configuration.
    </ResponseField>

    <ResponseField name="status" type="string">
      Will be `"disabled"` after this operation completes.
    </ResponseField>

    <ResponseField name="copy_percentage" type="string">
      Percentage of the trader's order size that had been configured to copy.
    </ResponseField>

    <ResponseField name="copy_buy" type="boolean">
      Whether buy copying had been enabled.
    </ResponseField>

    <ResponseField name="copy_sell" type="boolean">
      Whether sell copying had been enabled.
    </ResponseField>

    <ResponseField name="reverse_copy" type="boolean">
      Whether trade direction inversion had been enabled.
    </ResponseField>

    <ResponseField name="min_price" type="string">
      Minimum price filter that had been in effect.
    </ResponseField>

    <ResponseField name="max_price" type="string">
      Maximum price filter that had been in effect.
    </ResponseField>

    <ResponseField name="slippage" type="string">
      Buy-side slippage tolerance that had been in effect.
    </ResponseField>

    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp when this configuration was originally created.
    </ResponseField>

    <ResponseField name="updated_at" type="string">
      ISO 8601 timestamp when this configuration was last updated (i.e., the time of this disable operation).
    </ResponseField>
  </Expandable>
</ResponseField>

