Use this endpoint to retrieve a single copytrade configuration by its ID. This is useful for inspecting the current settings of a specific configuration before updating it, or for verifying the outcome of a previous create or update operation. The API returns `COPYTRADE_CONFIG_NOT_FOUND` when the configuration does not exist or belongs to a different account. Requires the `copytrade:read` scope.

## Request

```
GET /copytrade/configs/{config_id}
```

### Required Scope

`copytrade:read`

### Path Parameters

<ParamField path="config_id" type="string" required>
  The unique identifier of the copytrade configuration to retrieve.
</ParamField>

## Response

The response `data` field contains a full `CopytradeConfig` object.

<ResponseField name="data" type="object">
  <Expandable title="CopytradeConfig fields">
    <ResponseField name="config_id" type="string">
      Unique identifier for this copytrade configuration.
    </ResponseField>

    <ResponseField name="target_address" type="string">
      The trader wallet address being followed.
    </ResponseField>

    <ResponseField name="tag" type="string">
      Optional human-readable label for this configuration.
    </ResponseField>

    <ResponseField name="status" type="string">
      Current status of the configuration. One of `running`, `paused`, or `disabled`.
    </ResponseField>

    <ResponseField name="copy_percentage" type="string">
      Percentage of the trader's order size being copied.
    </ResponseField>

    <ResponseField name="copy_fixed_amount" type="string">
      Fixed token amount per copied trade, if set instead of a percentage.
    </ResponseField>

    <ResponseField name="max_token_spend" type="string">
      Maximum token spend cap per trade.
    </ResponseField>

    <ResponseField name="total_token_spend" type="string">
      Cumulative token spend cap across all copied trades.
    </ResponseField>

    <ResponseField name="min_copy_amount_per_trade" type="string">
      Minimum copied amount per trade.
    </ResponseField>

    <ResponseField name="max_copy_amount_per_trade" type="string">
      Maximum copied amount per trade.
    </ResponseField>

    <ResponseField name="min_price" type="string">
      Minimum token price filter.
    </ResponseField>

    <ResponseField name="max_price" type="string">
      Maximum token price filter.
    </ResponseField>

    <ResponseField name="slippage" type="string">
      Buy-side market order slippage tolerance.
    </ResponseField>

    <ResponseField name="copy_buy" type="boolean">
      Whether buy trades from the target are copied.
    </ResponseField>

    <ResponseField name="copy_sell" type="boolean">
      Whether sell trades from the target are copied.
    </ResponseField>

    <ResponseField name="limit_order_copy" type="boolean">
      Whether buy limit orders from the target are copied.
    </ResponseField>

    <ResponseField name="sell_limit_order_copy" type="boolean">
      Whether sell limit orders from the target are copied.
    </ResponseField>

    <ResponseField name="reverse_copy" type="boolean">
      Whether the copied trade direction is inverted.
    </ResponseField>

    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp when this configuration was created.
    </ResponseField>

    <ResponseField name="updated_at" type="string">
      ISO 8601 timestamp when this configuration was last updated.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Codes

| Code | Description |
|---|---|
| `COPYTRADE_CONFIG_NOT_FOUND` | The configuration with the given `config_id` does not exist or belongs to another account. |

## Example Request

```bash
curl -X GET "https://test-eu.polycopy.fun/openapi/v1/copytrade/configs/123" \
  -H "X-API-Key: pk_your_key_id" \
  -H "X-Timestamp: 1717200000" \
  -H "X-Nonce: abc123" \
  -H "X-Signature: <signature>"
```
