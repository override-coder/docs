sidebarTitle: "Market Types"
description: "Enable or disable market type filters for a copytrade config. Use 1 to enable and 2 to disable each market type. Omit fields to leave them unchanged."
---

This endpoint controls which market categories are copied for a given copytrade configuration. You can enable or disable any combination of market types in a single request — send only the fields you want to change and omit the rest to leave them as-is. Requires the `copytrade:manage` scope. You may optionally supply an `Idempotency-Key` header to safely retry the request.

## Request

```
PATCH /copytrade/configs/{config_id}/market-types
```

### Required Scope

`copytrade:manage`

### Optional Header

| Header | Description |
|---|---|
| `Idempotency-Key` | A unique key (e.g. a UUID) you generate. Repeated requests with the same key return the original response without applying the change a second time. |

### Path Parameters

<ParamField path="config_id" type="string" required>
  The unique identifier of the copytrade configuration whose market type filters you want to update.
</ParamField>

### Request Body

All fields except `config_type` are optional. For each market type field, use `1` to enable it or `2` to disable it. Omit a field entirely to leave its current setting unchanged.

<ParamField body="config_type" type="integer" required>
  Specifies the configuration type. Use `1` for an application config or `2` for a template config.
</ParamField>

<ParamField body="all" type="integer">
  Set all market types at once. `1` enables all, `2` disables all. Setting this field overrides individual market type fields in the same request.
</ParamField>

<ParamField body="sports" type="integer">
  Sports prediction markets. `1` = enabled, `2` = disabled.
</ParamField>

<ParamField body="crypto" type="integer">
  All crypto markets (across all intervals). `1` = enabled, `2` = disabled.
</ParamField>

<ParamField body="crypto_m5" type="integer">
  5-minute crypto markets. `1` = enabled, `2` = disabled.
</ParamField>

<ParamField body="crypto_m15" type="integer">
  15-minute crypto markets. `1` = enabled, `2` = disabled.
</ParamField>

<ParamField body="crypto_h1" type="integer">
  1-hour crypto markets. `1` = enabled, `2` = disabled.
</ParamField>

<ParamField body="crypto_h4" type="integer">
  4-hour crypto markets. `1` = enabled, `2` = disabled.
</ParamField>

<ParamField body="crypto_btc" type="integer">
  BTC crypto markets. `1` = enabled, `2` = disabled.
</ParamField>

<ParamField body="crypto_eth" type="integer">
  ETH crypto markets. `1` = enabled, `2` = disabled.
</ParamField>

<ParamField body="crypto_sol" type="integer">
  SOL crypto markets. `1` = enabled, `2` = disabled.
</ParamField>

<ParamField body="crypto_xrp" type="integer">
  XRP crypto markets. `1` = enabled, `2` = disabled.
</ParamField>

<ParamField body="politics" type="integer">
  Politics prediction markets. `1` = enabled, `2` = disabled.
</ParamField>

<ParamField body="weather" type="integer">
  Weather prediction markets. `1` = enabled, `2` = disabled.
</ParamField>

<ParamField body="other" type="integer">
  Uncategorized markets not covered by the above types. `1` = enabled, `2` = disabled.
</ParamField>

## Example Request

The following example enables all crypto markets and disables sports markets for configuration `123`:

```bash
curl -X PATCH "https://test-eu.polycopy.fun/openapi/v1/copytrade/configs/123/market-types" \
  -H "X-API-Key: pk_your_key_id" \
  -H "X-Timestamp: 1717200000" \
  -H "X-Nonce: abc123" \
  -H "X-Signature: <signature>" \
  -H "Content-Type: application/json" \
  -d '{"config_type": 1, "crypto": 1, "sports": 2}'
```
