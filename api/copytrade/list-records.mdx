---
title: "GET /copytrade/records — List Copytrade Execution Records"
sidebarTitle: "Records"
description: "Retrieve copytrade execution records for your account. Filter by config_id or status. Returns cursor-paginated results with filled and failed trades."
---

Copytrade records capture every trade execution attempt made by your copytrade configurations. Each record reflects a single trade that was triggered by a target trader's activity, and includes the outcome, pricing, amounts, and — when a trade fails — a machine-readable error message. Use this endpoint to audit trade history, diagnose failures, and monitor the performance of your configurations. Requires the `copytrade:read` scope.

## Request

```
GET /copytrade/records
```

### Required Scope

`copytrade:read`

### Query Parameters

<ParamField query="config_id" type="integer">
  Filter results to records belonging to a specific copytrade configuration.
</ParamField>

<ParamField query="status" type="integer">
  Filter results by execution status. See the status table below for valid values.
</ParamField>

<ParamField query="limit" type="integer" default="50">
  Number of results to return per page. Must be between `1` and `200`.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor from a previous response's `next_cursor` field. Omit to start from the first page.
</ParamField>

### Record Status Values

| Status | Integer Value | Description |
|---|---|---|
| Success | `1` | The trade was executed successfully and fully filled. |
| Failed | `2` | The trade execution failed. Check the `error_msg` field for the failure reason. |
| Waiting | `3` | The trade is pending execution. |
| Canceled | `4` | The trade was canceled before execution. |
| Partial fill | `5` | The trade was executed but only partially filled. |

## Response

<ResponseField name="code" type="string">
  Response status code string, e.g. `"OK"`.
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable status message.
</ResponseField>

<ResponseField name="request_id" type="string">
  Unique identifier for this API request, useful for support and debugging.
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="data fields">
    <ResponseField name="items" type="array">
      Array of execution record objects.

      <Expandable title="Record fields">
        <ResponseField name="record_id" type="string">
          Unique identifier for this execution record.
        </ResponseField>

        <ResponseField name="config_id" type="integer">
          The ID of the copytrade configuration that triggered this trade.
        </ResponseField>

        <ResponseField name="order_id" type="string">
          The on-chain order ID associated with the copied trade.
        </ResponseField>

        <ResponseField name="status" type="integer">
          Execution status integer. See the status table above.
        </ResponseField>

        <ResponseField name="error_msg" type="string">
          Machine-readable error code describing why the trade failed. Only present when `status` is `2` (failed).
        </ResponseField>

        <ResponseField name="market" type="string">
          The market address or identifier in which the trade was executed.
        </ResponseField>

        <ResponseField name="target_price" type="string">
          The price at which the target trader's original order was placed.
        </ResponseField>

        <ResponseField name="target_amount" type="string">
          The amount of the target trader's original order.
        </ResponseField>

        <ResponseField name="copy_price" type="string">
          The price at which your copied order was placed.
        </ResponseField>

        <ResponseField name="copy_amount" type="string">
          The amount of your copied order.
        </ResponseField>

        <ResponseField name="side" type="string">
          Trade direction: `"buy"` or `"sell"`.
        </ResponseField>

        <ResponseField name="create_dt" type="integer">
          Unix timestamp (seconds) when this record was created.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="next_cursor" type="string | null">
      Cursor to pass as the `cursor` query parameter to retrieve the next page. `null` when you have reached the last page.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

The following example retrieves failed execution records for configuration `123`:

```bash
curl -X GET "https://test-eu.polycopy.fun/openapi/v1/copytrade/records?config_id=123&status=2" \
  -H "X-API-Key: pk_your_key_id" \
  -H "X-Timestamp: 1717200000" \
  -H "X-Nonce: abc123" \
  -H "X-Signature: <signature>"
```
