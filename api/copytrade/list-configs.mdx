---
title: "GET /copytrade/configs — List Copytrade Configurations"
sidebarTitle: "List Configs"
description: "List all copytrade configurations for your account with cursor-based pagination. Returns an empty items array when no configs exist."
---

This endpoint returns all copytrade configurations associated with your account, paginated using opaque cursors. You can use it to inspect current configurations, audit settings, or build dashboards that display trader-following activity. When no configurations exist, the `items` array is returned empty. Requires the `copytrade:read` scope.

## Request

```
GET /copytrade/configs
```

### Required Scope

`copytrade:read`

### Query Parameters

<ParamField query="limit" type="integer" default="50">
  Number of results to return per page. Must be between `1` and `200`.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor from a previous response's `next_cursor` field. Omit this parameter to start from the first page.
</ParamField>

## Response

<ResponseField name="code" type="string">
  Response status code string, e.g. `"OK"`.
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable status message.
</ResponseField>

<ResponseField name="request_id" type="string">
  Unique identifier for this API request, useful for support and debugging.
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="data fields">
    <ResponseField name="items" type="array">
      Array of `CopytradeConfig` objects. Empty when no configurations exist.

      <Expandable title="CopytradeConfig fields">
        <ResponseField name="config_id" type="string">
          Unique identifier for this copytrade configuration.
        </ResponseField>

        <ResponseField name="target_address" type="string">
          The trader wallet address being followed.
        </ResponseField>

        <ResponseField name="tag" type="string">
          Optional human-readable label for this configuration.
        </ResponseField>

        <ResponseField name="status" type="string">
          Current status of the configuration. One of `running`, `paused`, or `disabled`.
        </ResponseField>

        <ResponseField name="copy_percentage" type="string">
          Percentage of the trader's order size to copy, e.g. `"100"`.
        </ResponseField>

        <ResponseField name="copy_buy" type="boolean">
          Whether buy trades from the target are copied.
        </ResponseField>

        <ResponseField name="copy_sell" type="boolean">
          Whether sell trades from the target are copied.
        </ResponseField>

        <ResponseField name="reverse_copy" type="boolean">
          When `true`, the copied trade direction is inverted (buys become sells and vice versa).
        </ResponseField>

        <ResponseField name="min_price" type="string">
          Minimum token price filter. Trades below this price are skipped.
        </ResponseField>

        <ResponseField name="max_price" type="string">
          Maximum token price filter. Trades above this price are skipped.
        </ResponseField>

        <ResponseField name="slippage" type="string">
          Buy-side market order slippage tolerance.
        </ResponseField>

        <ResponseField name="created_at" type="string">
          ISO 8601 timestamp when this configuration was created.
        </ResponseField>

        <ResponseField name="updated_at" type="string">
          ISO 8601 timestamp when this configuration was last updated.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="next_cursor" type="string | null">
      Cursor to pass as the `cursor` query parameter to retrieve the next page. `null` when you have reached the last page.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash
curl -X GET "https://test-eu.polycopy.fun/openapi/v1/copytrade/configs?limit=10" \
  -H "X-API-Key: pk_your_key_id" \
  -H "X-Timestamp: 1717200000" \
  -H "X-Nonce: abc123" \
  -H "X-Signature: <signature>"
```
