---
title: "PATCH /copytrade/configs/{config_id} — Update Copytrade Config"
sidebarTitle: "Update Config"
description: "Update fields on an existing copytrade configuration. All fields are optional; omitted fields are unchanged. Boolean fields represent the desired final state."
---

Use this endpoint to update one or more fields on an existing copytrade configuration. All request body fields are optional — only send the fields you want to change. Fields you omit are left at their current values. Requires the `copytrade:manage` scope. You may optionally supply an `Idempotency-Key` header to safely retry the request.

## Request

```
PATCH /copytrade/configs/{config_id}
```

### Required Scope

`copytrade:manage`

### Optional Header

| Header | Description |
|---|---|
| `Idempotency-Key` | A unique key (e.g. a UUID) you generate. Repeated requests with the same key return the original response without applying the update a second time. |

### Path Parameters

<ParamField path="config_id" type="string" required>
  The unique identifier of the copytrade configuration to update.
</ParamField>

### Request Body

<ParamField body="target_address" type="string">
  Update the trader wallet address to follow.
</ParamField>

<ParamField body="tag" type="string">
  Update the human-readable label for this configuration.
</ParamField>

<ParamField body="copy_percentage" type="string">
  Update the percentage of the trader's order size to copy.
</ParamField>

<ParamField body="copy_fixed_amount" type="string">
  Update the fixed token amount per copied trade.
</ParamField>

<ParamField body="max_token_spend" type="string">
  Update the maximum token spend cap per trade.
</ParamField>

<ParamField body="total_token_spend" type="string">
  Update the cumulative token spend cap across all copied trades.
</ParamField>

<ParamField body="min_copy_amount_per_trade" type="string">
  Update the minimum copied amount per trade.
</ParamField>

<ParamField body="max_copy_amount_per_trade" type="string">
  Update the maximum copied amount per trade.
</ParamField>

<ParamField body="min_price" type="string">
  Update the minimum token price filter.
</ParamField>

<ParamField body="max_price" type="string">
  Update the maximum token price filter.
</ParamField>

<ParamField body="slippage" type="string">
  Update the buy-side market order slippage tolerance.
</ParamField>

<ParamField body="copy_buy" type="boolean">
  Set to `true` to copy buy trades, `false` to stop copying them.
</ParamField>

<ParamField body="copy_sell" type="boolean">
  Set to `true` to copy sell trades, `false` to stop copying them.
</ParamField>

<ParamField body="limit_order_copy" type="boolean">
  Set to `true` to copy buy limit orders, `false` to stop copying them.
</ParamField>

<ParamField body="sell_limit_order_copy" type="boolean">
  Set to `true` to copy sell limit orders, `false` to stop copying them.
</ParamField>

<ParamField body="reverse_copy" type="boolean">
  Set to `true` to invert copied trade direction, `false` to copy direction as-is.
</ParamField>

<ParamField body="status" type="string">
  Update the configuration status. One of `running`, `paused`, or `disabled`.
</ParamField>

<Note>
  `keyword_whitelist` and `keyword_blacklist` are rejected on PATCH requests. To add or remove keywords on an existing configuration, use `POST /copytrade/configs/{config_id}/keywords` to add a keyword and `DELETE /copytrade/configs/{config_id}/keywords/{key_word_id}` to remove one.
</Note>

<Tip>
  Boolean fields like `copy_buy`, `copy_sell`, and `reverse_copy` represent the **desired final state**, not a toggle. For example, sending `"copy_buy": true` ensures buy copying is enabled regardless of its current value — you do not need to read the current state first.
</Tip>

## Response

The response `data` field contains the full updated `CopytradeConfig` object reflecting all current field values after the patch is applied.

<ResponseField name="data" type="object">
  <Expandable title="CopytradeConfig fields">
    <ResponseField name="config_id" type="string">
      Unique identifier for this copytrade configuration.
    </ResponseField>

    <ResponseField name="target_address" type="string">
      The trader wallet address being followed.
    </ResponseField>

    <ResponseField name="tag" type="string">
      Human-readable label for this configuration.
    </ResponseField>

    <ResponseField name="status" type="string">
      Current status: `running`, `paused`, or `disabled`.
    </ResponseField>

    <ResponseField name="copy_percentage" type="string">
      Percentage of the trader's order size being copied.
    </ResponseField>

    <ResponseField name="copy_buy" type="boolean">
      Whether buy trades from the target are copied.
    </ResponseField>

    <ResponseField name="copy_sell" type="boolean">
      Whether sell trades from the target are copied.
    </ResponseField>

    <ResponseField name="reverse_copy" type="boolean">
      Whether the copied trade direction is inverted.
    </ResponseField>

    <ResponseField name="min_price" type="string">
      Minimum token price filter in effect.
    </ResponseField>

    <ResponseField name="max_price" type="string">
      Maximum token price filter in effect.
    </ResponseField>

    <ResponseField name="slippage" type="string">
      Buy-side slippage tolerance in effect.
    </ResponseField>

    <ResponseField name="updated_at" type="string">
      ISO 8601 timestamp when this configuration was last updated.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash
curl -X PATCH "https://test-eu.polycopy.fun/openapi/v1/copytrade/configs/123" \
  -H "X-API-Key: pk_your_key_id" \
  -H "X-Timestamp: 1717200000" \
  -H "X-Nonce: abc123" \
  -H "X-Signature: <signature>" \
  -H "Content-Type: application/json" \
  -d '{"copy_percentage": "50", "status": "paused"}'
```
