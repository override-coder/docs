---
title: "GET /wallet/withdrawals/{withdraw_id} — Withdrawal Status"
sidebarTitle: "Get Withdrawal"
description: "Poll the status of a pending or completed Polygon withdrawal using its withdraw_id. Returns status, tx_hash, and full withdrawal details."
---

After initiating a withdrawal with `POST /wallet/withdraw`, use this endpoint to poll its status until it reaches a terminal state of `succeeded` or `failed`. The response includes the on-chain transaction hash once the withdrawal has been submitted to the network. You must authenticate with a key that carries the `wallet:read` scope.

## Request

```
GET https://test-eu.polycopy.fun/openapi/v1/wallet/withdrawals/{withdraw_id}
```

### Required Scope

`wallet:read`

### Path Parameters

<ParamField path="withdraw_id" type="string" required>
  The unique withdrawal identifier returned by `POST /wallet/withdraw` (e.g. `wd_202606011200000001`).
</ParamField>

## Response

All responses use the standard envelope: `code`, `message`, `data`, and `request_id`. The `data` object contains the fields described below.

<ResponseField name="withdraw_id" type="string">
  The unique identifier for this withdrawal.
</ResponseField>

<ResponseField name="status" type="string">
  The current status of the withdrawal. Possible values:

  | Value | Meaning |
  |-------|---------|
  | `processing` | The withdrawal has been received and is being prepared. |
  | `submitting` | The withdrawal is being submitted to the blockchain. |
  | `submitted` | The transaction has been broadcast to the network; awaiting confirmation. |
  | `succeeded` | The withdrawal completed successfully on-chain. |
  | `failed` | The withdrawal could not be completed. |
</ResponseField>

<ResponseField name="network" type="string">
  The blockchain network used for this withdrawal. Always `polygon` in v1.
</ResponseField>

<ResponseField name="token" type="string">
  The contract address of the token that was withdrawn.
</ResponseField>

<ResponseField name="amount" type="string">
  The amount withdrawn, expressed as a decimal string.
</ResponseField>

<ResponseField name="destination" type="string">
  The target Polygon wallet address to which the funds were sent.
</ResponseField>

<ResponseField name="tx_hash" type="string | null">
  The on-chain transaction hash. Populated once the withdrawal reaches `submitted` or `succeeded`. `null` for withdrawals still in earlier states.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the withdrawal was created.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the most recent status update.
</ResponseField>

<Tip>
  Poll this endpoint every few seconds after creating a withdrawal. Use exponential backoff (e.g. 2 s, 4 s, 8 s, …) to avoid hitting rate limits. Stop polling once `status` is either `succeeded` or `failed`.
</Tip>

## Example Request

```bash
curl -X GET "https://test-eu.polycopy.fun/openapi/v1/wallet/withdrawals/wd_202606011200000001" \
  -H "X-API-Key: pk_your_key_id" \
  -H "X-Timestamp: 1717200000" \
  -H "X-Nonce: abc123" \
  -H "X-Signature: <signature>"
```
