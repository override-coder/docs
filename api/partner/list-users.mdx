---
title: "GET /partner/users — List Provisioned Partner Users"
sidebarTitle: "List Users"
description: "Retrieve a paginated list of users provisioned under your partner account. Returns users ordered newest first with cursor-based pagination."
---

This endpoint returns all users provisioned under your partner API key, ordered from newest to oldest. Use it to audit your user base, check wallet initialization states, or build a management dashboard. You must authenticate with a partner API key that carries the `partner:users:manage` or `*` scope.

## Request

```
GET https://test-eu.polycopy.fun/openapi/v1/partner/users
```

### Required Scope

`partner:users:manage` or `*`

### Query Parameters

<ParamField query="limit" type="integer" default="50">
  Number of user records to return per page. Maximum value is `200`.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor. Pass the `next_cursor` value from the previous response to fetch the next page. Omit this parameter to start from the beginning.
</ParamField>

## Response

All responses use the standard envelope: `code`, `message`, `data`, and `request_id`. The `data` object contains the fields described below.

<ResponseField name="items" type="array">
  List of partner user objects, ordered newest first.

  <Expandable title="User object fields">
    <ResponseField name="partner_account_id" type="string">
      The account ID of the partner that provisioned this user.
    </ResponseField>

    <ResponseField name="user_account_id" type="string">
      The unique account ID assigned to this user by the platform.
    </ResponseField>

    <ResponseField name="external_user_id" type="string">
      Your stable internal identifier for this user, as supplied at creation time.
    </ResponseField>

    <ResponseField name="display_name" type="string">
      Human-readable name for the user.
    </ResponseField>

    <ResponseField name="email" type="string">
      Email address associated with the user account.
    </ResponseField>

    <ResponseField name="status" type="string">
      Account status. One of `active` or `disabled`.
    </ResponseField>

    <ResponseField name="wallet_status" type="string">
      Wallet initialization status. One of `pending`, `ready`, or `failed`.
    </ResponseField>

    <ResponseField name="wallet_created" type="boolean">
      Indicates whether a wallet has been created for this user.
    </ResponseField>

    <ResponseField name="wallet_initialized_at" type="string">
      ISO 8601 timestamp of when the wallet was successfully initialized. May be absent if the wallet is not yet ready.
    </ResponseField>

    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp of when the user record was created.
    </ResponseField>

    <ResponseField name="updated_at" type="string">
      ISO 8601 timestamp of the most recent update to the user record.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="next_cursor" type="string | null">
  Cursor to use in your next request to retrieve the following page. When this field is absent or `null`, you have reached the last page.
</ResponseField>

## Example Request

```bash
curl -X GET "https://test-eu.polycopy.fun/openapi/v1/partner/users?limit=20" \
  -H "X-API-Key: pk_your_key_id" \
  -H "X-Timestamp: 1717200000" \
  -H "X-Nonce: abc123" \
  -H "X-Signature: <signature>"
```
