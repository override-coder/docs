title: "GET /orders — List Account Orders"
sidebarTitle: "List Orders"
description: "Retrieve all orders for the authenticated account. Returns an empty items array when no orders match. Supports address and locale filters."
---

This endpoint returns all orders placed by the account associated with your API key. You can optionally filter results by wallet address or request localized market titles by specifying a locale. If no orders have been placed yet — or none match the applied filters — the response will contain an empty `items` array rather than an error. This endpoint requires the `trade:read` scope.

## Request

**`GET /orders`**

**Base URL:** `https://test-eu.polycopy.fun/openapi/v1`

**Required scope:** `trade:read`

### Authentication Headers

All requests must include the following HMAC-signed headers:

| Header | Type | Description |
|---|---|---|
| `X-API-Key` | string | Your API key identifier |
| `X-Timestamp` | string | Unix timestamp of the request |
| `X-Nonce` | string | Unique nonce string for this request |
| `X-Signature` | string | HMAC signature computed over the request |

### Query Parameters

<ParamField query="address" type="string">
  Filter the result set to orders associated with a specific wallet address. Useful when your API key is authorized across multiple addresses.
</ParamField>

<ParamField query="locale" type="string">
  A locale code used to return market titles in the specified language (e.g., `en`). When omitted, titles are returned in their default language.
</ParamField>

## Response

All responses use the standard envelope. On success, the `data` object contains the following field:

<ResponseField name="items" type="array">
  A list of order objects. Returns an empty array when no orders match. Each object in the array contains the following fields:

  <Expandable title="Order object fields">
    <ResponseField name="order_id" type="string">
      The unique identifier for this order.
    </ResponseField>

    <ResponseField name="status" type="string">
      The current status of the order. See the [Order Statuses](#order-statuses) table for possible values.
    </ResponseField>

    <ResponseField name="asset" type="string">
      The asset or token identifier this order was placed against.
    </ResponseField>

    <ResponseField name="side" type="string">
      The order direction: `buy` or `sell`.
    </ResponseField>

    <ResponseField name="price" type="string | null">
      The limit price at which the order was placed, as a decimal string. `null` for market orders.
    </ResponseField>

    <ResponseField name="size" type="string | null">
      The asset quantity of the order. `null` if not yet determined.
    </ResponseField>

    <ResponseField name="created_at" type="string | null">
      The ISO 8601 timestamp indicating when the order was created. `null` if unavailable.
    </ResponseField>
  </Expandable>
</ResponseField>

## Order Statuses

| Status | Description |
|---|---|
| `open` | The order is active and pending fill. |
| `filled` | The order was fully executed. |
| `canceled` | The order was canceled before it could be fully filled. |
| `failed` | The order encountered an error during processing. |
| `unknown` | The order status could not be determined. |

## Example

### List All Orders

```bash
curl -X GET "https://test-eu.polycopy.fun/openapi/v1/orders?locale=en" \
  -H "X-API-Key: pk_your_key_id" \
  -H "X-Timestamp: 1717200000" \
  -H "X-Nonce: abc123" \
  -H "X-Signature: <signature>"
```

**Response:**

```json
{
  "code": "OK",
  "message": "success",
  "data": {
    "items": [
      {
        "order_id": "0xorder_id_here",
        "status": "filled",
        "asset": "0xasset_address",
        "side": "buy",
        "price": "0.55",
        "size": "18.18",
        "created_at": "2024-06-01T12:00:00Z"
      }
    ]
  },
  "request_id": "req_202606011200000001"
}
```

### Empty Result

When no orders exist or none match the filter, you receive a successful response with an empty array:

```json
{
  "code": "OK",
  "message": "success",
  "data": {
    "items": []
  },
  "request_id": "req_202606011200000002"
}
```

<Note>
  A `200 OK` response with an empty `items` array is the expected behavior when the account has no orders or when no orders match the supplied filters. This is not an error condition.
</Note>
