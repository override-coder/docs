---
title: "GET /positions — List Account Positions"
sidebarTitle: "List Positions"
description: "Retrieve all open positions for the authenticated account. Filter by market or show only redeemable positions using optional query parameters."
---

This endpoint returns all positions held by the account linked to your API key. You can use the optional query parameters to narrow results to specific markets or to surface only positions that are eligible for redemption. If the account holds no positions — or none match your filters — the response returns an empty `items` array. This endpoint requires the `positions:read` scope.

## Request

**`GET /positions`**

**Base URL:** `https://test-eu.polycopy.fun/openapi/v1`

**Required scope:** `positions:read`

### Authentication Headers

All requests must include the following HMAC-signed headers:

| Header | Type | Description |
|---|---|---|
| `X-API-Key` | string | Your API key identifier |
| `X-Timestamp` | string | Unix timestamp of the request |
| `X-Nonce` | string | Unique nonce string for this request |
| `X-Signature` | string | HMAC signature computed over the request |

### Query Parameters

<ParamField query="address" type="string">
  Filter positions by wallet address. Useful when your API key is authorized across multiple addresses.
</ParamField>

<ParamField query="market" type="array of strings">
  One or more market IDs to filter by. Repeat the parameter for multiple values, for example: `?market=a&market=b`. Only positions belonging to the specified markets are returned.
</ParamField>

<ParamField query="redeemable" type="boolean">
  When set to `true`, only positions that are currently eligible for redemption are returned. Omit this parameter to return all positions regardless of redemption status.
</ParamField>

## Response

All responses use the standard envelope. On success, the `data` object contains the following field:

<ResponseField name="items" type="array">
  A list of position objects. Returns an empty array when no positions match. Each object in the array contains the following fields:

  <Expandable title="Position object fields">
    <ResponseField name="asset" type="string">
      The asset or token identifier representing this position.
    </ResponseField>

    <ResponseField name="condition_id" type="string">
      The condition ID that identifies the prediction market contract for this position.
    </ResponseField>

    <ResponseField name="outcome" type="string">
      The outcome label this position is associated with (e.g., `"Yes"` or `"No"`).
    </ResponseField>

    <ResponseField name="size" type="string">
      The quantity of the asset held in this position.
    </ResponseField>

    <ResponseField name="avg_price" type="string | null">
      The average entry price across all fills that contributed to this position, as a decimal string. `null` if unavailable.
    </ResponseField>

    <ResponseField name="current_price" type="string | null">
      The current market price for the asset, as a decimal string. `null` if unavailable.
    </ResponseField>

    <ResponseField name="value" type="string | null">
      The current total value of the position in USD, as a decimal string. `null` if unavailable.
    </ResponseField>

    <ResponseField name="pnl" type="string | null">
      The unrealized profit or loss for this position in USD, as a decimal string. A positive value indicates profit; a negative value indicates a loss. `null` if unavailable.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example

### List All Positions

```bash
curl -X GET "https://test-eu.polycopy.fun/openapi/v1/positions" \
  -H "X-API-Key: pk_your_key_id" \
  -H "X-Timestamp: 1717200000" \
  -H "X-Nonce: abc123" \
  -H "X-Signature: <signature>"
```

### List Redeemable Positions Only

```bash
curl -X GET "https://test-eu.polycopy.fun/openapi/v1/positions?redeemable=true" \
  -H "X-API-Key: pk_your_key_id" \
  -H "X-Timestamp: 1717200000" \
  -H "X-Nonce: abc123" \
  -H "X-Signature: <signature>"
```

**Response:**

```json
{
  "code": "OK",
  "message": "success",
  "data": {
    "items": [
      {
        "asset": "0xasset_address",
        "condition_id": "0xcondition_id",
        "outcome": "Yes",
        "size": "100",
        "avg_price": "0.55",
        "current_price": "0.65",
        "value": "65.00",
        "pnl": "10.00"
      }
    ]
  },
  "request_id": "req_202606011200000001"
}
```

### Filter by Market

To retrieve positions for specific markets, repeat the `market` query parameter:

```bash
curl -X GET "https://test-eu.polycopy.fun/openapi/v1/positions?market=0xmarket_a&market=0xmarket_b" \
  -H "X-API-Key: pk_your_key_id" \
  -H "X-Timestamp: 1717200000" \
  -H "X-Nonce: abc123" \
  -H "X-Signature: <signature>"
```

<Note>
  A `200 OK` response with an empty `items` array is the expected behavior when the account holds no positions or when none match the supplied filters. This is not an error condition.
</Note>

<Tip>
  Use `redeemable=true` in combination with a `market` filter to efficiently identify which of your positions in a specific market are ready to be redeemed after resolution.
</Tip>
