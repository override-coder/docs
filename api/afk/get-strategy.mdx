Retrieve a single AFK strategy by its ID. This endpoint returns the full strategy configuration including all price filters, risk parameters, and technical indicator settings. If the strategy does not exist or belongs to a different account, the response will contain `"code": "NOT_FOUND"`. Requires the `afk:read` scope.

## Request

```http
GET /afk/strategies/{strategy_id}
```

**Base URL:** `https://test-eu.polycopy.fun/openapi/v1`

**Required scope:** `afk:read`

<Note>
  All requests must include the four HMAC-signed authentication headers: `X-API-Key`, `X-Timestamp`, `X-Nonce`, and `X-Signature`.
</Note>

## Path Parameters

<ParamField path="strategy_id" type="string" required>
  The unique identifier of the AFK strategy to retrieve. This is the `config_id` returned by [List Strategies](/api/afk/list-strategies) or [Create Strategy](/api/afk/create-strategy).
</ParamField>

## Response Fields

The `data` object contains a single AFKStrategy with the following fields.

<ResponseField name="data" type="object">
  <Expandable title="AFKStrategy fields" defaultOpen>
    <ResponseField name="config_id" type="string">
      Unique strategy identifier.
    </ResponseField>
    <ResponseField name="status" type="string">
      Current strategy status. One of `running`, `paused`, or `disabled`.
    </ResponseField>
    <ResponseField name="tag" type="string">
      Human-readable label assigned to this strategy.
    </ResponseField>
    <ResponseField name="market" type="string">
      Market identifier the strategy trades on.
    </ResponseField>
    <ResponseField name="side" type="integer">
      Trade direction. `1` = Up (buy outcome), `2` = Down (sell outcome).
    </ResponseField>
    <ResponseField name="buy_amount" type="string">
      Buy amount per AFK order, expressed as a decimal string.
    </ResponseField>
    <ResponseField name="min_price" type="string">
      Minimum price threshold for order execution.
    </ResponseField>
    <ResponseField name="max_price" type="string">
      Maximum price threshold for order execution.
    </ResponseField>
    <ResponseField name="slippage" type="string">
      Accepted slippage percentage for market orders.
    </ResponseField>
    <ResponseField name="take_profit_percentage" type="string">
      Percentage gain that triggers a take-profit exit.
    </ResponseField>
    <ResponseField name="take_profit_price" type="string">
      Absolute price level that triggers a take-profit exit.
    </ResponseField>
    <ResponseField name="stop_loss_percentage" type="string">
      Percentage loss that triggers a stop-loss exit.
    </ResponseField>
    <ResponseField name="stop_loss_price" type="string">
      Absolute price level that triggers a stop-loss exit.
    </ResponseField>
    <ResponseField name="use_kdj" type="integer">
      KDJ indicator switch. `0` = off, `1` = on.
    </ResponseField>
    <ResponseField name="use_macd" type="integer">
      MACD indicator switch. `0` = off, `1` = on.
    </ResponseField>
    <ResponseField name="use_atr" type="integer">
      ATR indicator switch. `0` = off, `1` = on.
    </ResponseField>
    <ResponseField name="created_at" type="datetime | null">
      ISO 8601 timestamp of when the strategy was created. May be `null` for legacy strategies.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

| Code | Description |
|------|-------------|
| `NOT_FOUND` | The strategy does not exist or does not belong to your account. |

## Example Request

```bash
curl -X GET "https://test-eu.polycopy.fun/openapi/v1/afk/strategies/42" \
  -H "X-API-Key: pk_your_key_id" \
  -H "X-Timestamp: 1717200000" \
  -H "X-Nonce: abc123" \
  -H "X-Signature: <signature>"
```
