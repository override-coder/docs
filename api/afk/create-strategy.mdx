Create a new AFK automated trading strategy. Once created and set to `running`, the strategy will execute trades automatically based on your configured rules — including price range filters, technical indicator signals, and profit/loss triggers. Newly created strategies are returned with a `status` of `paused`; you must update the status to `running` to activate automated trading. Requires the `afk:manage` scope.

## Request

```http
POST /afk/strategies
```

**Base URL:** `https://test-eu.polycopy.fun/openapi/v1`

**Required scope:** `afk:manage`

<Note>
  All requests must include the four HMAC-signed authentication headers: `X-API-Key`, `X-Timestamp`, `X-Nonce`, and `X-Signature`.
</Note>

**Optional header:** `Idempotency-Key` — supply a unique value (e.g., a UUID) to safely retry the request without creating duplicate strategies.

## Request Body

<ParamField body="market" type="string" required>
  Market identifier or market group the strategy will trade on (e.g., `"crypto_btc"`).
</ParamField>

<ParamField body="side" type="integer" required>
  Trade direction. `1` = Up (buy outcome), `2` = Down (sell outcome).
</ParamField>

<ParamField body="buy_amount" type="string" required>
  Buy amount per AFK order, expressed as a decimal string (e.g., `"10"`).
</ParamField>

<ParamField body="tag" type="string">
  User-visible label for the strategy. Appears in list and record responses.
</ParamField>

<ParamField body="period" type="integer">
  Trading period or window for the strategy's execution cycle.
</ParamField>

<ParamField body="time_start_sec" type="integer">
  Unix timestamp (seconds) for when the strategy's daily active window begins.
</ParamField>

<ParamField body="time_end_sec" type="integer">
  Unix timestamp (seconds) for when the strategy's daily active window ends.
</ParamField>

<ParamField body="min_price" type="string">
  Minimum price threshold. Orders will not be placed when the market price is below this value.
</ParamField>

<ParamField body="max_price" type="string">
  Maximum price threshold. Orders will not be placed when the market price is above this value.
</ParamField>

<ParamField body="price_delta_min" type="string">
  Minimum price delta filter. Used to restrict entries based on recent price movement.
</ParamField>

<ParamField body="price_delta_max" type="string">
  Maximum price delta filter. Used to restrict entries based on recent price movement.
</ParamField>

<ParamField body="limit_price" type="string">
  Limit order price. Only used when `limit_order_copy` is set to `1`.
</ParamField>

<ParamField body="order_expired_ttl_seconds" type="integer">
  Time-to-live in seconds for limit orders. The order is canceled if not filled within this window.
</ParamField>

<ParamField body="total_rounds" type="integer">
  Maximum number of trading rounds the strategy will execute before stopping automatically.
</ParamField>

<ParamField body="take_profit_percentage" type="string">
  Percentage gain that triggers an automatic take-profit exit (e.g., `"30"` = 30%).
</ParamField>

<ParamField body="take_profit_price" type="string">
  Absolute price level that triggers an automatic take-profit exit.
</ParamField>

<ParamField body="stop_loss_percentage" type="string">
  Percentage loss that triggers an automatic stop-loss exit (e.g., `"20"` = 20%).
</ParamField>

<ParamField body="stop_loss_price" type="string">
  Absolute price level that triggers an automatic stop-loss exit.
</ParamField>

<ParamField body="limit_order_copy" type="integer">
  Order type switch. `0` = market order (default), `1` = limit order.
</ParamField>

<ParamField body="slippage" type="string">
  Accepted slippage percentage for market orders (e.g., `"2"` = 2%). Only applies when `limit_order_copy` is `0`.
</ParamField>

<ParamField body="use_kdj" type="integer">
  KDJ indicator switch. `0` = off (default), `1` = on.
</ParamField>

<ParamField body="use_macd" type="integer">
  MACD indicator switch. `0` = off (default), `1` = on.
</ParamField>

<ParamField body="use_atr" type="integer">
  ATR indicator switch. `0` = off (default), `1` = on.
</ParamField>

<Tip>
  **Technical indicators:** KDJ, MACD, and ATR are standard technical analysis indicators. Enabling one or more of these adds additional market signal filtering to the strategy's entry rules — orders will only be placed when the chosen indicators confirm a valid entry condition. You can combine multiple indicators for stricter filtering.
</Tip>

## Example Request

```bash
curl -X POST "https://test-eu.polycopy.fun/openapi/v1/afk/strategies" \
  -H "X-API-Key: pk_your_key_id" \
  -H "X-Timestamp: 1717200000" \
  -H "X-Nonce: abc123" \
  -H "X-Signature: <signature>" \
  -H "Idempotency-Key: strategy-uuid-here" \
  -H "Content-Type: application/json" \
  -d '{
    "market": "crypto_btc",
    "side": 1,
    "buy_amount": "10",
    "tag": "BTC Up Strategy",
    "min_price": "0.1",
    "max_price": "0.9",
    "slippage": "2",
    "stop_loss_percentage": "20",
    "take_profit_percentage": "30",
    "use_kdj": 1
  }'
```
