Partner APIs are available only to approved partners. They use the same API Key + HMAC-SHA256 signing flow as the trading APIs, but the key must be a partner API key.

## Partner key requirements

Use a partner API key with the required scope:

| Operation | Required scope |
|---|---|
| `GET /partner/users` | `partner:users:manage` or `*` |
| `POST /partner/users` | `partner:users:manage` or `*` |
| `POST /partner/users/{user_account_id}/api-keys` | `partner:api_keys:manage` or `*` |

Partner API keys are issued by PolyCop. They are not created by the public API.

## Provision a user

Call `POST /partner/users` with your stable `external_user_id`. The endpoint is idempotent for the same partner and external user id: if the user already exists, it returns the existing user.

```json
{
  "external_user_id": "user_123",
  "display_name": "Alice",
  "email": "alice@example.com"
}
```

The response includes `user_account_id` and `wallet_status`.

```json
{
  "partner_account_id": "9912345678901234564821",
  "user_account_id": "9912345678901234564822",
  "external_user_id": "user_123",
  "status": "active",
  "wallet_status": "ready",
  "wallet_created": true,
  "created_at": "2026-07-09T07:00:00Z",
  "updated_at": "2026-07-09T07:00:00Z"
}
```

`wallet_status=ready` means the user's custodial trading wallet is initialized. If the status is `pending` or `failed`, retry `POST /partner/users` with the same `external_user_id` before creating a user API key.

## Create a user API key

After the partner user is ready, call `POST /partner/users/{user_account_id}/api-keys`.

```json
{
  "name": "user trading key",
  "scopes": ["trade:read", "trade:order", "wallet:read"]
}
```

The response includes `key_id` and `secret`. The `secret` is shown only once.

<Warning>
  Store the returned `secret` immediately. PolyCop cannot show it again.
</Warning>

If `scopes` is omitted, PolyCop grants the default user scopes: `trade:read`, `trade:order`, `trade:cancel`, `wallet:read`, `positions:read`, `copytrade:read`, `copytrade:manage`, `afk:read`, and `afk:manage`.

User API keys cannot use the wildcard `*` scope. `withdraw:create` is not supported for partner user API keys in v1 because withdrawals require account-level TOTP.
