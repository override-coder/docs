Partner APIs are available only to approved partners. They use the same API Key + HMAC-SHA256 signing flow as the trading APIs, but the key must be a partner API key.

## Partner key requirements

Use a partner API key with the required scope:

| Operation | Required scope |
|---|---|
| `GET /partner/users` | `partner:users:manage` or `*` |
| `POST /partner/users` | `partner:users:manage` or `*` |
| `POST /partner/users/{user_account_id}/api-keys` | `partner:api_keys:manage` or `*` |
| `DELETE /partner/users/{user_account_id}/api-keys/{key_id}` | `partner:api_keys:manage` or `*` |

Partner API keys are issued by PolyCop. Use them to create or get partner users, list those users, and create user API keys for a selected `user_account_id`.

## List partner users

Call `GET /partner/users` to list partner users generated for your partner account. The endpoint is paginated with `limit` and `cursor`.

```bash
GET /partner/users?limit=50
```

The response contains non-sensitive user information only.

```json
{
  "items": [
    {
      "partner_account_id": "9912345678901234564821",
      "user_account_id": "9912345678901234564822",
      "external_user_id": "user_123",
      "display_name": "Alice",
      "status": "active",
      "wallet_status": "ready",
      "wallet_address": "0xYourWalletAddress",
      "proxy_address": "0xYourProxyAddress",
      "created_at": "2026-07-09T07:00:00Z",
      "last_active_at": "2026-07-09T08:30:00Z"
    }
  ],
  "next_cursor": null
}
```

Use `next_cursor` as the next request's `cursor` until it is absent or `null`.

## Create a user API key

Call `POST /partner/users` with your stable `external_user_id`. If that user already exists for your partner account, PolyCop returns the existing user.

```json
{
  "external_user_id": "user_123",
  "display_name": "Alice",
  "email": "alice@example.com"
}
```

Then call `POST /partner/users/{user_account_id}/api-keys` with the returned `user_account_id`.

```json
{
  "name": "user trading key",
  "scopes": ["trade:read", "trade:order", "wallet:read"]
}
```

The response includes the user API key. The `secret` is shown only once.

<Warning>
  Store the returned `secret` immediately. PolyCop cannot show it again.
</Warning>

If `scopes` is omitted, PolyCop grants the default user scopes: `trade:read`, `trade:order`, `trade:cancel`, `wallet:read`, `positions:read`, `copytrade:read`, `copytrade:manage`, `afk:read`, and `afk:manage`.

User API keys cannot use the wildcard `*` scope. `withdraw:create` is not supported for partner user API keys in v1 because withdrawals require account-level TOTP.

## Revoke a user API key

Call `DELETE /partner/users/{user_account_id}/api-keys/{key_id}` to revoke a user API key created by your partner account.

```bash
DELETE /partner/users/9912345678901234564822/api-keys/pk_xxx
```

Partners can only revoke user API keys created by their own partner account. This endpoint cannot revoke keys created by another partner or direct user API keys.
