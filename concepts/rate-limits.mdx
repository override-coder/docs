PolyCop OpenAPI applies several rate-limit buckets to each request. A request must pass every bucket that applies to it.

These are the default production limits. A deployment can override them in configuration.

## Shared limits

Every API Key + HMAC request is checked before and after authentication.

| Bucket | Limit | Applies to |
|---|---:|---|
| Global | 6,000 requests / minute | All OpenAPI traffic |
| Global burst | 1,500 requests / 10 seconds | All OpenAPI traffic |
| IP | 60 requests / minute | Source IP |
| IP burst | 20 requests / 10 seconds | Source IP |
| Public endpoint | 120 requests / minute | HTTP method + endpoint path |
| Public endpoint burst | 40 requests / 10 seconds | HTTP method + endpoint path |
| Account | 120 requests / minute | Authenticated account |
| Account burst | 40 requests / 10 seconds | Authenticated account |
| API key | 120 requests / minute | API key id |
| API key burst | 40 requests / 10 seconds | API key id |

## Rate-limit response

When a request exceeds a limit, the API returns HTTP `429` with `code=RATE_LIMITED`.

| Header | Description |
|---|---|
| `Retry-After` | Seconds to wait before retrying |
| `X-RateLimit-Limit` | The limit for the bucket that rejected the request |
| `X-RateLimit-Remaining` | Remaining requests in the current window, usually `0` |
| `X-RateLimit-Reset` | Seconds until the current bucket resets |

Use the `Retry-After` value from the response. Do not retry immediately with a new signature.
